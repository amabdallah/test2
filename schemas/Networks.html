Schema: Networks

Tables

Table NameComment
ConnectionsThe Connections table lists each Instance that has the topology link and specifies a flow direction by referencing the start and end node instances the link connects.
InstancesAn instance is a particular realization or implementation of an Object Type for a Data Structure in a specific location. The Instance concept helps modelers to replicate a distinct Object Type to represent specific water system components in their models with particular data values.
MappingIt is the central table in WaM-DaM which maps out data values to their attributes, metadata, and organizing structure (i.e., MasterNetworks, Scenarios, ObjectTypes, Instances, and DataStructure). Therefore the MetadataMapping table is often used to query data from WaM-DaM database.
MasterNetworksA Master Network is a specific implementation of a Data Structure, its Instances, and data values within a spatial location (e.g., Bear River Watershed, Utah). Master Networks help researchers to organize and then search groups of Instances that belong to a Data Structure in space. For example, a Data Structure for a WEAP model can be used to create separate WEAP model for the Bear River and the Weber River watersheds in Utah.
ScenarioMappingThe ScenarioMapping is a bridge table that allows associating many MetadataMapping records with many scenarios. This table allows to map out the duplicate topology, metadata, and data between scenarios
ScenariosA scenario organizes different topologies or connections of Instances in a Master Network and different metadata or data values for the attributes that describe those Instances. A Scenario concept helps users represent different modeling alternatives that simulate socio-economic changes in water resources networks and their data
